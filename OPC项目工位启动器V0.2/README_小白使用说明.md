# OPC 工位启动器 V0.2 小白使用说明

这个版本只做一件事：先进入独立 OPC 项目工位，再打开项目窗口。  
如果不能确认已经进入独立工位，它会直接停止，不会把网页、Finder、文档开到你当前正在工作的桌面。

## 第一次安装

1. 双击 `install.command`。
2. 如果提示要安装免费组件 AeroSpace，点“安装”。
3. 如果 macOS 提示给 AeroSpace 辅助功能权限，按提示允许。
4. 安装完成后，桌面会出现唯一入口：`OPC开工.command`。

以后只需要双击桌面的 `OPC开工.command`。

## 每次开工

双击 `OPC开工.command` 后，启动器会按这个顺序做：

1. 检查 AeroSpace 是否可用。
2. 切换或创建名为 `OPC` 的独立工作区。
3. 确认当前已经在 `OPC` 工作区。
4. 如果 `OPC` 工作区已有窗口，只切回去，不重复打开。
5. 如果是空工作区，再打开新的独立 Chrome 项目窗口和 Finder 项目窗口。

Chrome 会使用独立资料目录，不会往你原来的 Chrome 窗口里加标签页。

## 会自动创建哪些文件夹

位置是：

`~/AI工位/OPC小红书封面工具/`

里面会有：

- `00_入口`
- `01_商品资料_链接截图`
- `02_小红书参考案例`
- `03_封面素材`
- `04_Canva导出`
- `05_GPT输出文案`
- `06_最终成品`
- `07_Codex_未来开发`
- `08_归档`

你不需要记这个路径，开工后 Finder 会自动打开到这里。

## 怎么换 ChatGPT 链接或加链接

打开安装目录里的 `config.json`，修改 `links`：

```json
{
  "name": "新的链接名字",
  "url": "https://example.com/"
}
```

要换 ChatGPT 项目对话，就同时改：

- `chatgptUrl`
- `links` 里名字为 `ChatGPT OPC 项目对话` 的 `url`

改完后再次双击 `OPC开工.command`。

## 失败怎么办

如果弹窗说“未能进入独立项目工位”，这是正常的保护机制。  
意思是：启动器没有把项目窗口开到你当前桌面，它已经停止了。

常见原因：

- 没安装 AeroSpace。
- AeroSpace 第一次打开，还没给辅助功能权限。
- Google Chrome 没安装。
- `config.json` 写错了。

把弹窗内容发给 Ryan 处理即可。

## 怎么卸载

双击 `uninstall.command`。

卸载只删除启动器和桌面入口，不会删除你的 OPC 项目资料。项目资料仍保留在：

`~/AI工位/OPC小红书封面工具/`

## 重要说明

`OPC开工.command` 是 macOS 的命令文件，双击时系统会短暂打开一个 Terminal 窗口。启动器会尽快把真正工作放到后台执行，并且项目窗口只会在通过工作区验证后打开。
