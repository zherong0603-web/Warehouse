# OPC macOS 项目工位启动器 V0.2 技术说明

## 核心结论

V0.2 不再使用简单 `open URL`、`open Finder` 或默认打开文档。  
它采用“安全门”设计：只有在能确认已经切换到独立项目工作区后，才打开 Chrome 和 Finder。否则直接失败。

## 为什么不用原生 Mission Control 作为默认方案

macOS 没有公开 API 可以稳定完成这些动作：

- 创建 Space。
- 给 Space 命名。
- 判断某个项目 Space 是否已经存在。
- 稳定切回指定 Space。
- 在打开窗口前验证当前 Space 就是目标项目 Space。

AeroSpace 官方文档也说明 Apple 没有提供公开 API 来操作 Spaces，所以 AeroSpace 自己实现了 Workspaces。V0.2 因此把原生 Mission Control 自动化视为不满足 P0 安全标准的方案。

参考：

- AeroSpace 安装方式：https://nikitabobko.github.io/AeroSpace/guide#installation
- AeroSpace Workspaces 说明：https://nikitabobko.github.io/AeroSpace/guide#emulation-of-virtual-workspaces
- AeroSpace 命令说明：https://nikitabobko.github.io/AeroSpace/commands

## 当前安全后端

默认后端：`aerospace`

用到的命令：

- `aerospace list-workspaces --focused`
- `aerospace workspace OPC`
- `aerospace list-windows --workspace OPC --count`

启动器必须先通过这些检查：

1. `config.json` 可读取。
2. `spaceBackend` 是 `aerospace`。
3. Google Chrome 存在。
4. AeroSpace CLI 存在。
5. AeroSpace 可返回当前工作区。
6. `aerospace workspace OPC` 执行成功。
7. `list-workspaces --focused` 确认当前工作区是 `OPC`。

任意一步失败，脚本调用 `safe_fail`，弹窗提示并退出，不打开项目网页、Finder 或文档。

## Chrome 隔离策略

脚本不会使用：

```bash
open https://...
```

而是使用独立资料目录：

```bash
open -n -a "Google Chrome" --args \
  --user-data-dir="$WORKSPACE_ROOT/00_入口/.chrome-profile" \
  --no-first-run \
  --new-window \
  "file:///.../OPC工位首页.html" \
  "https://chatgpt.com/..." \
  "https://www.canva.com/"
```

这样不会把 ChatGPT、Canva 加到用户当前 Chrome 窗口。第一次使用时，这个独立 Chrome 资料目录需要单独登录。

## Finder 策略

脚本先切换到 `OPC` 工作区，再用 AppleScript 创建 Finder 新窗口：

```applescript
tell application "Finder"
    make new Finder window to targetFolder
    activate
end tell
```

如果无法先确认工作区，Finder 不会被打开。

## 文档策略

V0.2 不再直接打开 `.md` 文件。  
每次启动会生成：

```text
00_入口/OPC工位首页.html
00_入口/今日开工记录_YYYY-MM-DD.md
```

实际打开的是 HTML 首页，并且它作为项目 Chrome 新窗口里的第一个标签页出现。

## 重复点击策略

进入 `OPC` 工作区后，脚本会检查：

```bash
aerospace list-windows --workspace OPC --count
```

如果已有窗口，说明项目工位已经打开，本次只切回工作区，不重复打开新的 Chrome/Finder。

## 可测试命令

这些命令不会打开项目窗口：

```bash
OPC_LAUNCHER_FOREGROUND=1 OPC_LAUNCHER_NO_DIALOG=1 ./OPC开工.command --dry-run
OPC_LAUNCHER_FOREGROUND=1 OPC_LAUNCHER_NO_DIALOG=1 ./OPC开工.command --prepare-only
OPC_LAUNCHER_FOREGROUND=1 OPC_LAUNCHER_NO_DIALOG=1 ./OPC开工.command --simulate-space-failure
```

`--simulate-space-failure` 用于验收“新建桌面失败时保护现场”：它必须弹失败提示并退出，不打开 Chrome、Finder 或文档。

## 已知边界

`OPC开工.command` 是 macOS 命令文件，双击时系统会短暂创建 Terminal 启动窗口。脚本会把实际启动流程放到后台，并尝试关闭这个启动窗口。若要做到完全没有启动窗口，下一版应交付 `.app` 或登录项/菜单栏小程序，但这超出了本次“只要一个 command 入口”的交付范围。
