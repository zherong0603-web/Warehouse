# OPC 通用化样例

## 1. B2B 线索质量复盘模板

### 输入

- 线索来源：内容、投放、转介绍、私域、展会、客服。
- 客户原话：必须保留原文片段。
- 业务字段：平台 / 行业、市场、品类、规模、当前方案、供应商、痛点、时间计划。
- 组织反馈：销售、客服、交付、报价、产品系统的补充信息。

### 输出

- 绿灯：有明确场景、规模、痛点、推进意向和证据。
- 黄灯：方向相关但字段不足。
- 红灯：证据显示无效、低质量或非目标客户。
- 灰灯：证据不足，不能判断。

### 固定判断句式

- 判断：绿灯 / 黄灯 / 红灯 / 灰灯。
- 原因：基于哪些事实。
- 证据片段：引用客户原话或内部反馈。
- 缺失字段：下一步判断还缺什么。
- 下一步问谁：责任方。
- 下一步问题：可直接发出去的问题。

## 2. B2B 组织缺口雷达模板

### 缺口类型

- 前端线索字段缺口：客户信息不够，无法判断质量。
- 销售推进缺口：客户有意向但缺数据包或决策链。
- 交付能力缺口：服务边界、SLA、禁限品、实施条件不清。
- 报价模型缺口：缺少计费项、阶梯报价、成本拆分。
- 产品系统缺口：看板、数据、状态同步能力不清。
- 投放归因缺口：内容 / 渠道无法关联到线索质量。
- 管理优先级缺口：不知道优先服务哪类客户。

### 输出字段

- 缺什么信息。
- 责任方。
- 为什么影响判断。
- 补齐后能用于什么。
- 是否能转成内容选题。
- 来源证据。

## 3. B2B 内容实验模板

### 内容实验不是普通选题

普通选题只回答“写什么”。内容实验必须回答：

- 目标客户是谁。
- 对应哪个真实痛点。
- 对应哪项业务能力。
- 为什么现在值得测。
- 预期吸引什么 MQL。
- 发布前需要谁确认。
- 上线后看什么指标。
- 来源证据是什么。

### 推荐字段

- 实验 ID。
- 实验标题。
- 目标客户。
- 对应痛点。
- 对应能力。
- 素材形式。
- 为什么值得测。
- 预期吸引的 MQL 类型。
- 发布前需要谁确认。
- 上线后看什么指标。
- 来源证据。

## 4. B2B 作战日报模板

### 个人作战日报

用于一线增长负责人，必须能指导今天的动作：

- 一句话结论。
- 今日 MQL 质量概览。
- 今日最值得盯的线索。
- 今日最大组织缺口。
- 今日最值得做的内容实验。
- 今天应该问谁什么。
- 今天最该做的 3 个动作。
- 可沉淀到 OPC 的资产。

### 管理层摘要

用于老板或业务负责人，必须简洁、业务化、不甩锅：

- 今日 MQL 质量概览。
- 高价值 MQL 特征。
- 低质量 / 信息不足原因。
- 内容实验方向。
- 需要协同事项。
- 管理判断。

## 5. 适用行业

适用于客单价较高、销售周期较长、交付链路复杂、内容获客需要反复验证线索质量的 B2B 行业，例如：

- 跨境物流 / 海外仓 / 供应链服务。
- SaaS / 企业软件。
- 工业设备 / 自动化设备。
- 企业咨询 / 专业服务。
- 招商加盟 / 渠道业务。
- 医疗器械、实验室服务等需要销售跟进的专业服务。

## 6. 不适用场景

不适用于：

- 低客单价、即时成交、几乎不需要销售判断的业务。
- 纯品牌曝光项目，短期不关心线索质量。
- 没有客户原话、没有销售反馈、没有任何证据输入的场景。
- 已经有成熟 CRM 和 BI 闭环，且不需要内容实验反馈的团队。
- 要求系统替用户做商业决策，但不提供证据和业务资料的场景。

## 7. 从 YQN 抽象出的通用方法

YQN 的具体闭环是美国仓内容 -> 私域线索 -> MQL 复盘 -> 销售反馈 -> 组织缺口 -> 内容实验。抽象到 OPC，可复用为：

业务内容 -> 线索进入 -> 证据抽取 -> 质量分层 -> 组织补洞 -> 下一轮内容 / 销售实验 -> 日报复盘 -> 方法论沉淀。
