# README_小白说明

## 这是做什么的

这是“YQN 美国仓增长闭环系统”的阶段 1 离线 MVP。它不接飞书、不接 CRM、不抓全网，只读取本地输入文件，并自动生成 6 个业务输出。

## 你每天怎么用

1. 打开项目目录 `YQN美国仓增长闭环系统/`。
2. 把当天资料放进 `00_输入收件箱/`。
3. 只放这 3 类文件：`.txt`、`.md`、`.csv`。
4. 双击 `运行阶段1.command`。
5. 等终端提示“阶段 1 离线闭环已完成”。
6. 查看 6 个输出文件。

默认双击运行时，系统会跳过文件名或目录名中带“样例”的资料，避免把示例数据混进真实日报。

## 会生成哪些文件

最新输出会固定更新在：

- `01_事实提取/事实提取表.csv`
- `02_MQL质量复盘/MQL质量复盘.md`
- `03_组织缺口雷达/组织缺口雷达.md`
- `04_内容实验库/内容实验库.csv`
- `05_作战日报/个人作战日报.md`
- `05_作战日报/管理层摘要.md`

每次运行还会自动保存一份带日期时间的副本：

- `08_验收包/阶段1运行记录/YYYY-MM-DD_HHMMSS/`

## 样例怎么跑

样例输入放在：

- `00_输入收件箱/样例_阶段1/`

如果只想跑样例，在项目目录打开终端执行：

```bash
python3 run_stage1.py --input 00_输入收件箱/样例_阶段1
```

## 这个工具不会做什么

- 不接飞书。
- 不接 CRM / 企微生产系统。
- 不读取 PDF、Word、Excel、PPT。
- 不做全网爬虫。
- 不让用户手动打分。
- 不围绕墨西哥仓展开。
- 不生成普通新闻日报。
- 不承诺没有证据的价格、SLA、仓库能力或系统能力。

## 没有输入文件怎么办

不会崩溃。系统会提示“未发现输入文件”，并把信息写进 `logs/`。

## 判断原则

系统会自动识别绿灯、黄灯、红灯、灰灯：

- 绿灯：美国仓场景明确，关键信息较完整。
- 黄灯：有美国仓或海外仓需求，但关键字段不足。
- 红灯：证据显示低质量，例如泛咨询、无店无货、只要开店教程。
- 灰灯：证据不足，不能判断。灰灯不是低质量。

所有判断都必须有证据片段；没有证据就写“待确认”。
