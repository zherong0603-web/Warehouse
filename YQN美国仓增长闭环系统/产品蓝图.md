# 产品蓝图

## 1. 系统解决什么业务问题

YQN 美国仓增长闭环系统解决的是“小红书获客之后，线索质量无法被稳定复盘”的问题。用户每天会收到小红书私信、评论、投放反馈、客服转述、销售反馈、会议纪要和产品资料，但这些信息通常分散在多个聊天窗口和表格里，难以回答三个关键问题：

1. 今天来的 MQL 是不是美国仓目标客户。
2. 哪些素材和选题更容易吸引高质量客户。
3. 组织内部还缺哪些信息，导致报价、跟进或内容复盘无法推进。

系统的目标不是替代 CRM，而是把非结构化资料整理成能行动的增长复盘。

## 2. 为什么核心是增长闭环

如果只做信息总结，输出很容易变成“今天有几条线索”的流水账。用户真正需要的是闭环：

素材 / 内容 / 渠道 -> 私域开口 -> MQL 判定 -> 销售反馈 -> 客户痛点 -> 组织缺口 -> 内容实验 -> 作战日报 -> 下一轮素材优化。

闭环的价值在于：每一条客户原话都能回到内容和投放实验，每一次销售反馈都能反向修正 MQL 判断，每一个组织缺口都能变成下一步追问或素材验证。

## 3. 用户每天怎么用

用户只需要把资料丢进 `00_输入收件箱/`，例如：

- 小红书私信或评论截图转写。
- 私域 / 客服聊天记录。
- 销售跟进反馈。
- 投放素材标题、封面、评论和投流结果。
- 仓库、报价、运营、产品系统提供的能力边界信息。

系统读取资料后输出：

- 事实提取表：先把可验证事实抽出来。
- MQL 质量复盘：自动给绿灯、黄灯、红灯、灰灯，不要求用户手动打分。
- 组织缺口雷达：指出缺什么信息、谁该补、补齐后用于什么。
- 内容实验库：把客户痛点转成下一轮小红书实验。
- 个人作战日报：告诉用户今天该干什么。
- 管理层摘要：给强哥 / Alice 看，简洁、业务化、不甩锅。
- OPC 通用化模板：把 YQN 的经验抽象成 B2B 增长方法。

## 4. Codex 每阶段怎么交付

阶段 0：交付闭环样例产品原型。重点是验证业务结构是否成立，输出样例必须像真实美国仓业务，并且每个判断有证据。

阶段 1：开发本地可运行的最小处理流程。输入一批 Markdown / CSV / 文本资料，自动生成事实提取、MQL 复盘、组织缺口、内容实验和日报。

阶段 2：增加配置化规则和模板。让灯号判断、YQN 能力映射、日报模板可以调整，但不要求用户手工评分。

阶段 3：接入更稳定的数据入口。可以考虑飞书文档、表格或固定收件箱，但暂不做复杂 Bot。

## 5. ChatGPT 验收官怎么判断

验收官重点看：

- 业务方向是否聚焦美国仓 / 北美跨境电商。
- 输出是否能帮助用户判断 MQL 质量。
- 判断是否有证据片段。
- 红灯和灰灯是否区分清楚。
- 内容实验是否来自客户痛点，而不是空泛标题党。
- 管理层摘要是否能直接转给老板。
- OPC 模板是否可复用到其他 B2B 增长场景。

## 6. 当前不做什么

当前不做：

- 不做全网爬虫。
- 不做复杂飞书 Bot。
- 不接 CRM / 企微生产系统。
- 不让用户手动打标或手动评分。
- 不把墨西哥仓作为重点方向。
- 不承诺未被资料证明的仓库能力、价格、时效或系统功能。
- 不生成泛泛的公司宣传稿。
